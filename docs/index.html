Documentation v1.0

CRF Annotation Editor

A desktop tool that automatically extracts fields from clinical CRF PDFs, maps them to SDTM variables using a RAW-to-SDTM Excel mapping workbook, and lets you review, correct, and export fully annotated PDFs — all from a native window.

PDF Pipeline

Five-step automated pipeline extracts CRF fields, matches them against SDTM mappings, and annotates your PDF with coloured overlay boxes.

Confidence Scoring

TF-IDF–based engine scores each annotation: ≥ 80 % auto-resolves, 60–79 % flags for review, below 60 % marks unmapped.

Manual Review

Drag to draw new boxes, right-click to edit, add or remove from the review queue, and apply SDTM labels with the AI-powered suggestion panel.

Analytics

Live ring-chart shows resolution percentage. Sidebar counters display per-status counts for every domain — updated after every action.

PDF Export

Exports a pixel-perfect annotated PDF with all annotation boxes and SDTM labels rendered directly onto the original CRF pages.

Session Persistence

All annotations, undo history, and pipeline data are persisted per session under outputs/<session_id>/. Sessions are isolated and reloadable.

Excel Mapping File

Required. The pipeline will not run without assets/RAW_SDTM_Mappings.xlsx. Place this file before launching the application.

The annotation pipeline relies on a RAW-to-SDTM variable mapping workbook that defines how raw CRF field names are translated into standardised SDTM variables. The file must be located at exactly:

Automated-aCRF/
└── assets/
    └── RAW_SDTM_Mappings.xlsx   ← exact filename, exact location

Supported Formats

Format A — One sheet per SDTM domain

Each domain has its own sheet named by its domain code. Recognised domain codes:

CM AE DM DS EX LB VS MH PE QS SC SU EG FA PR
Format B — Single consolidated sheet

One sheet named "RAW-SDTM Mappings" containing all domains in a single table. The pipeline auto-detects which format is present.

Required Columns

Both formats must contain these five columns (exact names, case-insensitive; column order does not matter):

Column NameDescriptionExample
Source DatasetThe raw/source dataset nameCONMED
Source VariableThe raw CRF field nameCMTRT
SDTM DatasetThe target SDTM domain codeCM
SDTM VariableThe target SDTM variable nameCMTRT
SDTM LabelHuman-readable labelConcomitant Medication Treatment

Quick Start

From zero to an annotated PDF in five steps.

01

Place the mapping file

Copy your RAW_SDTM_Mappings.xlsx into assets/. Without it, the pipeline cannot run.

02

Launch the application

Run python app.py. The native window opens at 1440 × 900 px.

03

Upload your CRF PDF

Click "Upload PDF" in the side panel (or drag-and-drop). The PDF is rendered page-by-page in the central canvas.

04

Run the pipeline

Click "Run Pipeline". The five-step extraction and annotation pipeline runs automatically. A progress indicator appears during processing.

05

Review & Export

Work through the Review Queue, correct any NEEDS_REVIEW or UNMAPPED items, then click "Export PDF".

Every annotation box on the canvas is colour-coded by its status:

Green
RESOLVED / USER_CORRECTED — mapping confirmed
Amber
NEEDS_REVIEW — confidence 60–79 %, in the active queue
Red
UNMAPPED — confidence < 60 %, no match found
Grey

Violet
User-drawn annotation — newly created, not yet submitted to the pipeline

Side Panel

The collapsible left sidebar contains all session controls, organised into tabs.

Upload

Click "Upload PDF" or drag-and-drop a PDF file. The canvas renders the first page immediately. File name and page count are shown.

Pipeline

Click "Run Pipeline" to execute all five pipeline steps: text extraction, field identification, mapping lookup, confidence scoring, and box annotation. Progress is shown during the run.

Once complete, a summary table lists every field with its matched SDTM variable and confidence score.

Analytics

Displays a live animated ring chart showing the overall resolution percentage. Below it, per-status counters show the breakdown across RESOLVED, NEEDS_REVIEW, UNMAPPED, USER_CORRECTED, and NOT_SUBMITTED. The ring resets and animates from zero every time the percentage changes.

Review Queue

Contains two sub-lists: Active (UNMAPPED + NEEDS_REVIEW) and Resolved (USER_CORRECTED + NOT_SUBMITTED). Click any row to jump to that annotation on the canvas. Right-click a row for context actions.

Export

Click "Export PDF" to render an annotated copy of your CRF. The output is saved to outputs/<session_id>/CRF_Annotated.pdf. A file-save dialog lets you choose a destination.

PDF Canvas

The central pane renders the PDF page-by-page and overlays all annotation boxes. This is where you visually inspect and interact with annotations.

Navigation

  • Use the page controls (arrow buttons or the page number input) at the bottom of the canvas to move between pages.
  • Press ← / → to navigate pages via keyboard.
  • The current page and total page count are always visible in the status bar.

Annotation Boxes

Each coloured box represents one extracted CRF field. The box label shows the SDTM variable name (e.g. CM.CMTRT). Hover over a box to see its full details in a tooltip.

Drawing New Annotations

  • Click and drag on any blank area of the canvas to draw a new annotation box.
  • Release the mouse to open the Edit Panel for that new annotation.
  • New user-drawn annotations start as violet and are assigned a local ID starting with user_.

Running the Pipeline

The pipeline is a five-step automated process that turns a raw CRF PDF into a fully annotated output.

1

Text Extraction

PyMuPDF extracts all text blocks from each page of the PDF, preserving coordinates and font metadata.

2

Field Identification

Heuristic rules segment the extracted text blocks into candidate CRF field names, labels, and values.

3

Mapping Lookup

Each candidate field is looked up in the in-memory table built from RAW_SDTM_Mappings.xlsx. Exact matches are flagged immediately.

4

Confidence Scoring

For each field, TF-IDF cosine similarity assigns a confidence score. Scores ≥ 80 % → RESOLVED; 60–79 % → NEEDS_REVIEW; < 60 % → UNMAPPED.

5

PDF Annotation

PyMuPDF renders coloured boxes at each field's PDF coordinates. The annotated image is returned to the canvas renderer.

Pipeline results are saved to outputs/<session_id>/components/. Re-running the pipeline on the same session overwrites the previous results.

Working with Annotations

Correcting an Annotation

  1. Click a box on the canvas (or click its row in the Review Queue) to open the Edit Panel.
  2. Edit the SDTM Domain, Variable, and Label fields — or click a suggestion from the AI panel.
  3. Click Save. Status becomes USER_CORRECTED and the box turns green.

Drawing a New Annotation

  1. Click and drag on blank canvas space to draw a box.
  2. Release — the Edit Panel opens for the new annotation.
  3. Fill in the SDTM fields and click Save.

Marking as Not Submitted

Right-click a box → Mark as Not Submitted. The annotation turns grey and moves to the Resolved list. It will not appear in the exported PDF's review overlay but its box is still drawn.

Removing an Annotation

Right-click → Remove Annotation (red button). The action is undoable via Ctrl+Z.

Adding / Removing from Review Queue

  • Add to Review Queue: Right-click any resolved annotation → Add to Review Queue. Status becomes NEEDS_REVIEW.
  • Remove from Review Queue: Right-click a NEEDS_REVIEW annotation (on canvas or in the queue) → Remove from Review Queue. Status reverts to UNMAPPED.

Exporting

Export produces a pixel-perfect annotated PDF with all annotation boxes and labels rendered onto the original pages.

  1. Switch to the Export tab in the side panel.
  2. Click "Export PDF".
  3. Choose the save destination in the file dialog.
  4. The output is also auto-saved to outputs/<session_id>/CRF_Annotated.pdf.
Export renders the PDF at high resolution — processing time scales with page count. For large CRFs (> 50 pages) allow up to a minute.

Output Directory Structure

outputs/
└── <session_id>/
    ├── components/
    │   ├── annotation_data.json   ← All annotation records
    │   └── pipeline_summary.txt   ← Pipeline run log
    ├── editor_state.json          ← Saved undo/redo history
    └── CRF_Annotated.pdf          ← Exported annotated PDF

Session IDs are derived from the PDF filename plus a random 8-character hex suffix (e.g. STUDY001_CRF_a3f8b21c). Running the same PDF twice creates two independent sessions.

Undo / Redo

The editor maintains a full undo history for all annotation changes within a session.

ShortcutAction
Ctrl+ZUndo last annotation change
Ctrl+Shift+ZRedo
Ctrl+YRedo (alternate)
Undo history is persisted to outputs/<session_id>/editor_state.json on every change, so it survives application restarts.

Status Reference

Every annotation carries one of six statuses. Statuses control colour, queue membership, and export behaviour.

StatusColourSet byQueueMeaning
RESOLVED Green Pipeline (≥ 80 %) — Auto-matched with high confidence; no action needed
NEEDS_REVIEW Amber Pipeline (60–79 %) or user Active Probable match but warrants human verification
UNMAPPED Red Pipeline (< 60 %) or user Active No reliable SDTM match found; needs manual correction
USER_CORRECTED Green User (after editing & saving) Resolved Manually reviewed and confirmed by the user
NOT_SUBMITTED Grey User (right-click action) Resolved Field exists on the CRF but is excluded from the submission
REMOVED Invisible User (delete action) — Annotation deleted; no longer rendered or exported

Pipeline runs but no annotations appear Excel column names don't match Update column constants in config.py

ValueError: No sheet named 'RAW-SDTM Mappings' Neither Format A nor B detected Rename consolidated sheet to RAW-SDTM Mappings or use per-domain sheets
PermissionError on outputs/ Insufficient write permissions Run from a directory where your user has write access
App window too small / content cut off OS display scaling > 100 % Set display scaling to 100 % or resize manually
KeyError in pipeline for a column Column header has extra whitespace Strip whitespace from column headers in your Excel file
